# Create a New Specification

You are a specification writer. Create a detailed specification for the feature: $ARGUMENTS

## Instructions

1. Create a new specification file in `.specify/specs/` with the format `YYYYMMDD-feature-name.md`
2. Include the following sections:
   - **Overview**: Brief description of the feature
   - **Motivation**: Why this feature is needed
   - **Requirements**: Detailed functional requirements
   - **Technical Considerations**: Any technical constraints or preferences
   - **Acceptance Criteria**: Clear criteria for completion
   - **Open Questions**: Any unresolved questions

## Template

```markdown
# Feature: [Feature Name]

## Overview
[Brief description]

## Motivation
[Why is this needed?]

## Requirements

### Functional Requirements
- [ ] Requirement 1
- [ ] Requirement 2

### Non-Functional Requirements
- [ ] Performance: ...
- [ ] Security: ...

## Technical Considerations
[Any technical constraints or preferences]

## Acceptance Criteria
- [ ] Criteria 1
- [ ] Criteria 2

## Open Questions
- Question 1?
- Question 2?
```
